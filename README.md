# nuxnet-inference

`nuxnet-inference` is a CLI-focused 3D nuclei localization and segmentation inference package for volumetric microscopy images, built around a 3D U-Net backbone [2,3].
It is designed for high-throughput usage where nuclei are segmented as foreground instances and can be converted into centroid-level measurements for downstream quantification.

Scientifically, this project follows the NuMorph setting for whole-brain nuclei analysis in cleared tissue and light-sheet microscopy volumes [1].
Operationally, it is intended to plug into Nextflow workflows (nf-core style), in particular the `nf-core/lsmquant` pipeline context.

For reproducibility, the runtime follows the same technical direction used in the mlf-core framework [4,5]: explicit random-seed control plus an optional deterministic execution mode for PyTorch/CUDA (`--deterministic`) where backend support allows.
This makes repeatable inference behavior easier to enforce in automated pipeline execution.

From a model-family perspective, this repository currently provides U-Net-based segmentation for volumetric inference; modern convolutional backbones such as ConvNeXt [6] are relevant architectural context for future model evolution but are not the default architecture in this package.

This repository ships:

- an inference CLI: `nuxnet-pred`
- a production 3D U-Net model backend (ported from prior liver CT segmentation work)
- segmentation utilities (focal loss, IoU/accuracy metrics, and volume dataset loader)
- deterministic/reproducibility helpers under `nuxnet_inference_package.mlf_core`
- OME-TIFF I/O and optional nuclei post-processing (TSV output with centroid and voxel-size features)

## Quick start

### Python version support

This project is tested to run on Python 3.9 through 3.12.
For the best compatibility with current dependencies, we recommend Python 3.12.

Install locally:

```console
pip install .
```

Run with autogenerated dummy 3D input and random UNet3D weights:

```console
nuxnet-pred smoke-test --arch unet3d --output ./predictions/demo
```

Run deterministic smoke-test (mlf-core style runtime controls):

```console
nuxnet-pred smoke-test --arch unet3d --output ./predictions/demo_det --deterministic
```

Run with a custom OME-TIFF 3D input volume (with `ZYX` axes):

```console
nuxnet-pred predict --arch unet3d --input ./input_volume.ome.tiff --output ./predictions/demo
```

Run deterministic inference:

```console
nuxnet-pred predict --arch unet3d --input ./input_volume.ome.tiff --output ./predictions/demo_det --deterministic
```

Run inference and extract nuclei instances table (TSV):

```console
nuxnet-pred predict --arch unet3d --input ./input_volume.ome.tiff --output ./predictions/demo --postprocess-instances --nuclei-label 1 --neighbor-radius 2.0
```

Disable normalization if needed:

```console
nuxnet-pred predict --arch unet3d --input ./input_volume.ome.tiff --output ./predictions/demo --no-normalize-input
```

Run with a checkpoint:

```console
nuxnet-pred predict --arch unet3d --classes 3 --in-channels 1 --model ./checkpoint.pt --input ./input_volume.ome.tiff --output ./predictions/demo
```

## CLI overview

The `nuxnet-pred` CLI now exposes two explicit subcommands:

- `predict`: run inference from one OME-TIFF volume or from all TIFF/OME-TIFF files in a folder.
- `smoke-test`: generate a random 3D volume, initialize an untrained model, run a forward pass, and write the predicted mask.

Both commands save masks as OME-TIFF files (`.ome.tiff`) with `ZYX` axes metadata. When `--postprocess-instances` is enabled in `predict`, a nuclei table is also written as `<output>.nuclei.tsv`. The post-processing uses Scikit-Image for mask handling, a SciPy KDTree (radius-based neighbors), and a NetworkX directed voxel graph with strongly connected components for clustering.

## Post-processing for nuclei localization (TSV export)

To execute post-processing, run `predict` with `--postprocess-instances`:

```console
nuxnet-pred predict --arch unet3d --input ./input_volume.ome.tiff --output ./predictions/demo --postprocess-instances --nuclei-label 1 --neighbor-radius 2.0
```

This writes:

- segmentation mask: `./predictions/demo.ome.tiff`
- nuclei table: `./predictions/demo.nuclei.tsv`

Algorithm summary (current implementation):

1. Build nuclei mask from predicted class label (`--nuclei-label`).
2. Extract nuclei voxel coordinates from the binary mask.
3. Build a SciPy KDTree and run radius range queries (`scipy.spatial.KDTree`, `query_ball_point`) to find nearby nuclei voxels.
4. Build a voxel topology graph in NetworkX (`networkx.DiGraph`) from neighbor relations.
5. Extract clusters using strongly connected components (`networkx.strongly_connected_components`).
6. Compute centroid `(z, y, x)` and `size_voxels` per component and export TSV.

Python libraries/functions used:

- Scikit-Image: `skimage.exposure.rescale_intensity` (optional input normalization), `skimage.measure.label` (mask labeling helper).
- SciPy: `scipy.spatial.KDTree`, `KDTree.query_ball_point`.
- NetworkX: `networkx.DiGraph`, `networkx.strongly_connected_components`.

## CLI parameters

Shared model/runtime parameters (available in both commands):

- `--arch [unet3d|dummy]`: model architecture.
- `--classes INTEGER`: number of segmentation classes (output channels).
- `--in-channels INTEGER`: number of input channels expected by the model.
- `--dropout-rate FLOAT`: dropout used in UNet3D blocks.
- `--seed INTEGER`: seed used for reproducibility.
- `--cuda/--no-cuda`: enable CUDA when available.
- `--deterministic/--no-deterministic`: toggle mlf-core-style deterministic runtime settings (CuDNN/CuBLAS/PyTorch deterministic algorithms).
- `--normalize-input/--no-normalize-input`: enable/disable input intensity normalization with `skimage.exposure.rescale_intensity` using image min/max.

`predict` parameters:

- `--input TEXT`: input OME-TIFF file (`.ome.tiff`/`.ome.tif`) or directory containing TIFF/OME-TIFF volumes. If omitted, random input is generated.
- `--model TEXT`: optional model checkpoint (plain `state_dict` or dict containing `state_dict`).
- `--output TEXT`: output prefix (single input) or output directory (input folder).
- `--input-shape TEXT`: shape `D,H,W` used only when `--input` is omitted.
- `--postprocess-instances/--no-postprocess-instances`: enable/disable nuclei instance extraction and TSV export.
- `--nuclei-label INTEGER`: class label interpreted as nuclei in the predicted mask (default `1`).
- `--neighbor-radius FLOAT`: radius used by KDTree to create a dense voxel-neighborhood graph (default `2.0`).

`smoke-test` parameters:

- `--output TEXT`: output prefix for the generated mask.
- `--input-shape TEXT`: random input shape `D,H,W`.

## How to run the smoke test

The command below runs the requested test workflow end-to-end:

1. Generates a random 3D input volume.
2. Initializes an untrained model.
3. Runs one forward pass.
4. Writes the segmentation mask as OME-TIFF.

```console
nuxnet-pred smoke-test --input-shape 8,16,16 --arch unet3d --classes 3 --in-channels 1 --output ./predictions/smoke_mask --no-cuda
```

This creates:

```text
./predictions/smoke_mask.ome.tiff
```

You can inspect command help at any time:

```console
nuxnet-pred --help
nuxnet-pred predict --help
nuxnet-pred smoke-test --help
```


## Reproducibility notes

- `--seed` controls RNG seeding for Python, NumPy, and Torch.
- `--deterministic` controls runtime algorithm selection and backend settings to maximize deterministic behavior where supported.
- Deterministic mode can be slower (`cudnn.benchmark=False`) and can raise runtime errors if an operation has no deterministic implementation in the current stack.
- Full reproducibility is strongest when software/hardware versions are fixed (PyTorch/CUDA/cuDNN/driver/GPU model).

## Notes

- `--arch dummy` remains available for smoke-testing.
- Checkpoints may be either a plain `state_dict` or a dict containing `state_dict`.
