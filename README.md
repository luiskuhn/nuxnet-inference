# nuxnet-inference

A lightweight PyTorch-only package for 3D segmentation inference.

This repository ships:

- an inference CLI: `nuxnet-pred`
- a production 3D U-Net model backend (ported from prior liver CT segmentation work)
- segmentation utilities (focal loss, IoU/accuracy metrics, and volume dataset loader)

## Quick start

### Python version support

This project is tested to run on Python 3.9 through 3.12.
For the best compatibility with current dependencies, we recommend Python 3.12.

Install locally:

```console
pip install .
```

Run with autogenerated dummy 3D input and random UNet3D weights:

```console
nuxnet-pred smoke-test --arch unet3d --output ./predictions/demo
```

Run with a custom OME-TIFF 3D input volume (with `ZYX` axes):

```console
nuxnet-pred predict --arch unet3d --input ./input_volume.ome.tiff --output ./predictions/demo
```

Run with a checkpoint:

```console
nuxnet-pred predict --arch unet3d --classes 3 --in-channels 1 --model ./checkpoint.pt --input ./input_volume.ome.tiff --output ./predictions/demo
```

## CLI overview

The `nuxnet-pred` CLI now exposes two explicit subcommands:

- `predict`: run inference from one OME-TIFF volume or from all TIFF/OME-TIFF files in a folder.
- `smoke-test`: generate a random 3D volume, initialize an untrained model, run a forward pass, and write the predicted mask.

Both commands save masks as OME-TIFF files (`.ome.tiff`) with `ZYX` axes metadata.

## CLI parameters

Shared model/runtime parameters (available in both commands):

- `--arch [unet3d|dummy]`: model architecture.
- `--classes INTEGER`: number of segmentation classes (output channels).
- `--in-channels INTEGER`: number of input channels expected by the model.
- `--dropout-rate FLOAT`: dropout used in UNet3D blocks.
- `--seed INTEGER`: seed used for reproducibility.
- `--cuda/--no-cuda`: enable CUDA when available.

`predict` parameters:

- `--input TEXT`: input OME-TIFF file (`.ome.tiff`/`.ome.tif`) or directory containing TIFF/OME-TIFF volumes. If omitted, random input is generated.
- `--model TEXT`: optional model checkpoint (plain `state_dict` or dict containing `state_dict`).
- `--output TEXT`: output prefix (single input) or output directory (input folder).
- `--input-shape TEXT`: shape `D,H,W` used only when `--input` is omitted.

`smoke-test` parameters:

- `--output TEXT`: output prefix for the generated mask.
- `--input-shape TEXT`: random input shape `D,H,W`.

## How to run the smoke test

The command below runs the requested test workflow end-to-end:

1. Generates a random 3D input volume.
2. Initializes an untrained model.
3. Runs one forward pass.
4. Writes the segmentation mask as OME-TIFF.

```console
nuxnet-pred smoke-test --input-shape 8,16,16 --arch unet3d --classes 3 --in-channels 1 --output ./predictions/smoke_mask --no-cuda
```

This creates:

```text
./predictions/smoke_mask.ome.tiff
```

You can inspect command help at any time:

```console
nuxnet-pred --help
nuxnet-pred predict --help
nuxnet-pred smoke-test --help
```

## Notes

- `--arch dummy` remains available for smoke-testing.
- Checkpoints may be either a plain `state_dict` or a dict containing `state_dict`.
